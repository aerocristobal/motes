# Configuration Reference

All configuration lives in `.memory/config.yaml`, generated by `mote init`. Every field is optional — missing values fall back to defaults. The file is loaded with `DefaultConfig()` as a base, then YAML values are overlaid on top.

## Section Reference

### scoring

Controls how motes are ranked when retrieved by `mote prime`, `mote context`, or graph traversal.

**Final score formula:**

```
score = (weight + edge_bonus + status_penalty + retrieval_bonus +
         salience_bonus + tag_specificity + interference) * recency_factor
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `edge_bonuses` | map | see above | Bonus per link type when mote is reached via that edge |
| `status_penalties` | map | `deprecated: -0.5` | Score penalty by mote status |
| `recency_decay.tiers` | list | 4 tiers | Time-based multiplier. Each tier has `max_days` (null = unbounded) and `factor` |
| `retrieval_strength.per_access_bonus` | float | `0.03` | Score boost per access |
| `retrieval_strength.max_bonus` | float | `0.15` | Cap on retrieval bonus |
| `salience` | map | see above | Bonus by mote origin |
| `explore_type_bonus` | float | `0.1` | Extra bonus for `explore` type motes |
| `tag_specificity.weight` | float | `0.2` | Weight of tag rarity in scoring |
| `tag_specificity.overload_threshold` | int | `15` | Tags with more motes than this are considered overloaded |
| `interference_per_contradiction` | float | `-0.1` | Penalty per active contradiction link |
| `max_results` | int | `12` | Maximum motes returned by traversal |
| `max_hops` | int | `2` | BFS hop limit from seed motes |
| `min_relevance_threshold` | float | `0.25` | Minimum score to include in results |

### priming

Controls ambient signal detection for `mote prime` and seed selection.

**Built-in signals:**

| Signal | What it does |
|--------|-------------|
| `git_branch` | Matches branch name against mote tags |
| `recent_files` | Parses recently modified file paths for keyword signals |
| `prompt_keywords` | Extracts keywords from user's prompt for tag matching |

**Custom co-access signals:**

```yaml
priming:
  signals:
    - name: git_branch
      type: built_in
    - name: recent_files
      type: built_in
    - name: prompt_keywords
      type: built_in
    - name: auth-deploy-coacccess
      type: co_access
      description: When working on auth, also surface deploy motes
      trigger_tags: [auth, authentication]
      boost_tags: [deploy, ci]
      boost_amount: 0.3
```

When any `trigger_tags` match the current context, motes with `boost_tags` get `boost_amount` added to their score.

### dream

Controls the headless LLM maintenance cycle.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `schedule_hint_days` | int | `2` | Suggested interval between dream runs |
| `provider.batch.model` | string | `claude-sonnet-4-20250514` | Model for batch analysis |
| `provider.reconciliation.model` | string | `claude-opus-4-20250514` | Model for cross-batch reconciliation |
| `batching.strategy` | string | `hybrid` | `hybrid` = 60% tag-clustered + 40% interleaved |
| `batching.max_motes_per_batch` | int | `10` | Maximum motes per LLM batch |
| `batching.clustered_fraction` | float | `0.6` | Fraction of motes in clustered batches |
| `batching.max_concurrent` | int | `4` | Maximum parallel batch invocations |
| `batching.self_consistency_runs` | int | `1` | Invoke each batch N times and keep majority-agreed visions. `1` = disabled, `3` = recommended. Mutually exclusive with `lens_mode`. |
| `batching.lens_mode.enabled` | bool | `false` | Enable lens mode: one run per lens instead of N identical runs. Mutually exclusive with `self_consistency_runs > 1`. |
| `batching.lens_mode.lenses` | []string | `[]` | Lens names to run. Valid: `structural`, `survivorship_bias`, `feedback_loops`, `confirmation_bias`, `inversion`, `probabilistic`, `first_principles`, `opportunity_cost`, `occams_razor` |
| `reconciliation.enabled` | bool | `true` | Whether to run Opus reconciliation pass |
| `reconciliation.max_refetch_motes` | int | `15` | Max motes to re-read during reconciliation |

**Pre-scan thresholds** control what the deterministic scanner flags:

| Field | Default | Description |
|-------|---------|-------------|
| `pre_scan.link_candidate_min_shared_tags` | `3` | Minimum shared tags to suggest a link |
| `pre_scan.staleness_threshold_days` | `180` | Days since last access before flagging stale |
| `pre_scan.tag_overload_threshold` | `15` | Mote count before a tag is considered overloaded |
| `pre_scan.theme_growth_threshold_pct` | `30` | Growth % to trigger constellation evolution |
| `pre_scan.compression_min_words` | `300` | Minimum body words to suggest compression |

| Field | Default | Description |
|-------|---------|-------------|
| `journal.max_tokens` | `2000` | Token budget for lucid log (cross-batch context) |
| `interrupts.high_severity_mote_pct` | `20` | % of motes with high-severity issues to trigger interrupt |

### strata

Controls the BM25-based reference knowledge search engine.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `chunking.strategy` | string | `heading-aware` | `heading-aware`, `function-level`, or `sliding-window` |
| `chunking.max_chunk_tokens` | int | `512` | Target chunk size in tokens |
| `chunking.overlap_tokens` | int | `50` | Overlap between sliding-window chunks |
| `retrieval.default_top_k` | int | `5` | Default number of results for strata queries |
| `retrieval.min_relevance_score` | float | `0.3` | Minimum BM25 score to include |
| `context_augment.enabled` | bool | `true` | Whether `mote context` also searches strata |
| `context_augment.max_augment_corpora` | int | `2` | Max corpora to search during context augmentation |
| `context_augment.chunks_per_corpus` | int | `3` | Chunks to include per corpus |

**Crystallization** controls when frequently-queried strata content should become a permanent mote:

| Field | Default | Description |
|-------|---------|-------------|
| `crystallization.min_queries` | `5` | Minimum query count before suggesting crystallization |
| `crystallization.min_sessions` | `3` | Minimum distinct sessions |
| `crystallization.staleness_threshold_days` | `180` | Days before flagging unused corpora |

## Sample Configurations

### Small Personal Project

Faster scoring with lower thresholds for a small knowledge base:

```yaml
scoring:
  max_results: 8
  max_hops: 1
  min_relevance_threshold: 0.15

dream:
  schedule_hint_days: 7
  reconciliation:
    enabled: false
  pre_scan:
    staleness_threshold_days: 365

strata:
  retrieval:
    default_top_k: 3
  context_augment:
    enabled: false
```

### Large Team Project

Tighter filtering and more aggressive maintenance for a large codebase with many contributors:

```yaml
scoring:
  max_results: 20
  max_hops: 3
  min_relevance_threshold: 0.35
  tag_specificity:
    overload_threshold: 25

priming:
  signals:
    - name: git_branch
      type: built_in
    - name: recent_files
      type: built_in
    - name: prompt_keywords
      type: built_in
    - name: frontend-api-coacccess
      type: co_access
      trigger_tags: [frontend, ui, react]
      boost_tags: [api, endpoints, schema]
      boost_amount: 0.2

dream:
  schedule_hint_days: 1
  batching:
    max_motes_per_batch: 15
    max_concurrent: 6
    self_consistency_runs: 3
  pre_scan:
    link_candidate_min_shared_tags: 2
    staleness_threshold_days: 90
    compression_min_words: 200

strata:
  chunking:
    max_chunk_tokens: 1024
  retrieval:
    default_top_k: 10
  context_augment:
    max_augment_corpora: 4
    chunks_per_corpus: 5
```

### Lens Mode (Mental Model Triangulation)

Munger's Rule of 3 Models applied to dream analysis — one psychological lens, one causal/economic lens, one structural lens:

```yaml
dream:
  batching:
    lens_mode:
      enabled: true
      lenses:
        - survivorship_bias    # psychological
        - feedback_loops       # causal
        - structural           # structural/mathematical
  reconciliation:
    enabled: true              # Opus synthesizes across lenses
```

When two or more lenses independently flag the same vision, that vision receives `CrossLensAgreement` — a stronger confidence signal than self-consistency voting, because the agreement comes from analytically distinct perspectives rather than identical prompts.

### Research/Writing Project

Emphasis on knowledge capture over task tracking. Longer staleness window, more generous results:

```yaml
scoring:
  edge_bonuses:
    builds_on: 0.4
    relates_to: 0.2
  recency_decay:
    tiers:
      - max_days: 30
        factor: 1.0
      - max_days: 180
        factor: 0.85
      - factor: 0.6
  max_results: 15
  min_relevance_threshold: 0.2

dream:
  schedule_hint_days: 3
  pre_scan:
    staleness_threshold_days: 365
    compression_min_words: 500

strata:
  chunking:
    strategy: heading-aware
    max_chunk_tokens: 768
  retrieval:
    default_top_k: 8
    min_relevance_score: 0.2
```

### Code-Heavy Project with Strata Focus

Function-level chunking for code corpora, aggressive strata integration:

```yaml
strata:
  chunking:
    strategy: function-level
    max_chunk_tokens: 256
    overlap_tokens: 0
  retrieval:
    default_top_k: 10
    min_relevance_score: 0.25
  context_augment:
    enabled: true
    max_augment_corpora: 3
    chunks_per_corpus: 5
  crystallization:
    min_queries: 3
    min_sessions: 2
```

### Minimal (Dream Disabled)

For environments without `claude` CLI access or when you only want manual operations:

```yaml
dream:
  schedule_hint_days: 0
  reconciliation:
    enabled: false

strata:
  context_augment:
    enabled: false
```
